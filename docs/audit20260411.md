# qwen-asr-provider 审计记录 2026-04-11

此文承接 [audit20260410.md](audit20260410.md) 与 [audit20260410-e2e.md](audit20260410-e2e.md)，只记今日复核后的现况。

## 已修正

### 1. realtime 文本不再无限单串累积

- 服务侧已改为 `recent_segments + live_tail` 视图。
- UI 改为 history/live 分层渲染，不再每次以全文大串重绘。
- 兼容字段仍保留，但主显示语义已转向 bounded display。

### 2. 全局 single-workload busy gate 已去除

- 旧实现以单一 `active_workload` 串行化 offline / chat / realtime / host capture。
- 现已移除此总锁。
- realtime 仍由 session 表与上限控制。
- host capture 仍保持单路约束。

### 3. async job 生命周期已补终态清理

- 旧实现中 async jobs 仅插入、查询、取消，无自然回收。
- 现已加入后台维护线程。
- 默认每 `60s` 扫描一次，清理超过 `3600s` 的终态 job。
- 对应 metrics 已补 cleanup run / evicted counters。

### 4. `/v1/realtime` 已补最小可用入口

- 现已提供单路 `/v1/realtime` JSON 控制面。
- 以 `session.create`、`input_audio_buffer.append`、`input_audio_buffer.commit` 三型事件映射既有 realtime 内核。
- 当前仍为 HTTP JSON 控制面，不是 WebSocket 双工实现。

## 仍存漂移

### 1. chat `stream=true` 仍非真增量

- 当前实现是先得完整结果，再拼成 SSE 样式输出。
- 这与 token-by-token 增量语义仍有差距。

### 2. runtime 骨架与 server 主路径仍分裂

- `SessionManager` / `TaskQueue` / `RealtimeSession` 等类型已存在。
- 主 HTTP 服务路径仍多直接落在 `SharedAsrModel` 与 server 内部状态上。
- 故“架构已具”与“主路径已用”不可混为一事。

### 3. inference 子层仍有 placeholder 性质

- `encoder.cc`、`decoder.cc`、`streaming_policy.cc` 仍偏轻骨架。
- 当前可跑主路径更多依赖 backend 中的直接推理实现。

### 4. trace / request_id 仍未贯通

- 现有日志与错误返回已可用。
- 但跨请求追踪字段仍未系统下沉至主要响应与日志链路。

## 风险判断

- 本轮所修两项属“高收益、低外溢”：
  - 去总锁，解除不必要跨接口互斥
  - 清终态 job，堵住常驻服务的累积泄漏面
- 其余未修项多属接口完备性与架构一致性问题，宜分批推进，不宜一轮并改。

## 建议次序

1. 将 chat streaming 改为真增量输出，再决定是否上 WS / 双工协议。
2. 逐步收束 runtime 骨架与 server 主路径，避免双轨长期并存。
3. 在 `/v1/realtime` 之上再补 WebSocket 双工面，而非继续扩私有 `/api/realtime/*`。
